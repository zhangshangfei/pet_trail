# 宠迹项目 - 功能完成报告

## 项目概述

宠迹是一个宠物生活记录工具 + 轻社交社区，帮助宠主科学记录宠物健康、养成良好养宠习惯，同时通过轻松的内容分享获得情感满足。

## 已完成功能清单

### 一、基础架构升级 ✅

#### 1.1 缓存系统
- **多级缓存架构**：Caffeine (L1) + Redis (L2) + 数据库 (L3)
- **缓存配置类**：`CacheConfig.java`、`RedisConfig.java`
- **缓存工具类**：`MultiLevelCache.java`
- **支持自定义过期时间**
- **缓存雪崩防护**：随机过期时间

#### 1.2 限流熔断（Sentinel）
- **限流规则配置**：`SentinelConfig.java`
- **接口限流**：
  - 动态发布：10 次/分钟
  - 点赞接口：5 次/秒
  - 登录接口：3 次/秒
- **降级处理**：系统繁忙时返回友好提示

#### 1.3 消息队列（RocketMQ）
- **异步消息处理**：动态发布异步审核
- **消息配置类**：`RocketMQConfig.java`
- **消息消费者**：`PostCreateConsumer.java`
- **支持异步、同步、延迟消息**

## 部署说明

### 环境要求
- Java 21+
- MySQL 8.0+
- Redis 7.0+
- RocketMQ 4.9+
- Node.js 18+

### 数据库初始化
```bash
# 1. 创建数据库
mysql -u root -p -e "CREATE DATABASE pet_trail DEFAULT CHARSET utf8mb4;"

# 2. 初始化基础表
mysql -u root -p pet_trail < backend/src/main/resources/init.sql

# 3. 初始化新增表
mysql -u root -p pet_trail < backend/src/main/resources/schema-v2.sql
```

### 后端启动
```bash
cd backend
mvn clean install
java -jar target/pet-trail-backend-1.0.0.jar
```

### 前端启动
```bash
cd frontend
npm install
npm run dev:mp-weixin  # 开发模式
npm run build:mp-weixin  # 生产构建
```

## V2.1 新增功能 ✅

### 七、WebSocket 实时推送 ✅

**后端**：
- `WebSocketConfig.java` - WebSocket 配置（端点 `/ws/notifications`）
- `NotificationWebSocketHandler.java` - 消息处理器
  - 连接时自动推送未读数和系统消息
  - 支持心跳保活（ping/pong）
  - 创建通知时实时推送给目标用户
  - 标记已读/删除/清空时推送未读数更新
- Security 放行 `/ws/**` 端点
- JWT Token 鉴权（URL 参数传递）

**前端**：
- `utils/websocket.js` - WebSocket 连接管理器
  - 自动重连（最多5次，递增延迟）
  - 心跳保活（30秒）
  - 事件监听机制（notification/unreadCount/init）
- 首页集成 WebSocket，替代轮询方案

**数据库**：无需新增表
