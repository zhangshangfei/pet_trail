# 宠迹项目启动说明

> 最后更新：2026年4月14日 | 版本：v2.1.0

## 项目目录

- 前端：`G:\openclaw_project\pet-trail\frontend`
- 后端：`G:\openclaw_project\pet-trail\backend`

---

## 快速启动

### 方式一：Docker Compose（推荐）

```powershell
# 一键启动所有服务（MySQL + Redis + 后端 + 前端）
cd G:\openclaw_project\pet-trail
docker-compose up -d
```

访问地址：
- 前端：http://localhost:8080
- 后端API：http://localhost:8080/api

### 方式二：手动启动

#### 1. 启动数据库和Redis

确保MySQL和Redis服务已启动：

```powershell
# 检查MySQL
mysql -u root -p -e "SELECT VERSION();"

# 检查Redis
redis-cli ping
```

#### 2. 初始化数据库

```powershell
# 执行数据库初始化脚本
cd G:\openclaw_project\pet-trail\backend
mysql -u root -p < src\main\resources\init.sql
```

#### 3. 启动后端

```powershell
# 进入后端目录
cd G:\openclaw_project\pet-trail\backend

# 启动Spring Boot
mvnw.cmd spring-boot:run
```

后端启动成功后，访问：http://localhost:8080/api/health

#### 4. 启动前端

打开新的命令行窗口：

```powershell
# 进入前端目录
cd G:\openclaw_project\pet-trail\frontend

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

前端启动成功后，访问：http://localhost:8080

## 数据库配置

后端配置文件：`backend/src/main/resources/application.yml`

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/pet_trail?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: root123654

  data:
    redis:
      host: localhost
      port: 6379
```

**注意事项**：
1. 确保MySQL服务已启动
2. 确保Redis服务已启动
3. 数据库`pet_trail`已创建并执行`init.sql`脚本
4. 用户名密码与配置一致

## 故障排查

### 前端启动失败

1. **检查Node.js版本**：
   ```powershell
   node -v
   ```
   确保版本 >= 18.0.0

2. **清除缓存重新安装**：
   ```powershell
   Remove-Item -Recurse -Force node_modules, package-lock.json
   npm install
   ```

3. **检查端口占用**：
   ```powershell
   netstat -ano | findstr "8080"
   ```
   如果端口被占用，修改`vite.config.ts`中的端口配置

4. **查看详细错误**：
   查看命令行输出的错误信息

### 后端启动失败

1. **检查Java版本**：
   ```powershell
   java -version
   ```
   确保版本 >= 21

2. **检查Maven版本**：
   ```powershell
   mvnw.cmd -v
   ```

3. **检查数据库连接**：
   - 确保MySQL服务运行：`Get-Service MySQL*`
   - 确保Redis服务运行：`redis-cli ping`
   - 检查`application.yml`中的数据库配置

4. **检查数据库是否初始化**：
   ```powershell
   mysql -u root -p -e "USE pet_trail; SHOW TABLES;"
   ```
   如果没有表，需要执行`init.sql`脚本

5. **查看详细日志**：
   关注控制台输出的错误信息，常见错误：
   - `Connection refused`：数据库连接失败
   - `BeanCreationException`：依赖注入失败
   - `Port already in use`：端口被占用

### 数据库连接失败

1. **检查MySQL服务**：
   ```powershell
   Get-Service MySQL*
   ```
   如果未运行，启动服务：
   ```powershell
   Start-Service MySQL*
   ```

2. **检查用户名密码**：
   确保`application.yml`中的用户名密码正确

3. **检查数据库是否存在**：
   ```powershell
   mysql -u root -p -e "SHOW DATABASES;"
   ```
   如果没有`pet_trail`，需要创建并初始化

### Redis连接失败

1. **检查Redis服务**：
   ```powershell
   redis-cli ping
   ```
   应该返回`PONG`

2. **启动Redis**：
   ```powershell
   redis-server
   ```
