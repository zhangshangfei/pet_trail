# 关注功能实现报告

## 实现概述

完成了首页Tab切换关注功能的后端逻辑实现，包括关注/取消关注、收藏持久化等功能。

## 完成的工作

### 1. 数据库表结构

#### 1.1 关注表 (follows)
```sql
CREATE TABLE IF NOT EXISTS `follows` (
  `id` bigint(20) NOT NULL AUTO_INCREMENT,
  `follower_id` bigint(20) NOT NULL COMMENT '关注者ID',
  `followee_id` bigint(20) NOT NULL COMMENT '被关注者ID',
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_follower_followee` (`follower_id`, `followee_id`),
  KEY `idx_follower_id` (`follower_id`),
  KEY `idx_followee_id` (`followee_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='关注表';
```

#### 1.2 收藏表 (post_ee)
```sql
CREATE TABLE IF NOT EXISTS `post_ee` (
  `id` bigint(20) NOT NULL AUTO_INCREMENT,
  `post_id` bigint(20) NOT NULL,
  `user_id` bigint(20) NOT NULL,
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_post_user` (`post_id`, `user_id`),
  KEY `idx_post_id` (`post_id`),
  KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='收藏表';
```

### 2. 后端实现

#### 2.1 实体类
- **Follow.java**: 关注实体，映射follows表
- **PostEe.java**: 收藏实体，映射post_ee表

#### 2.2 Mapper层
- **FollowMapper.java** + **FollowMapper.xml**: 
  - `deleteByFollowerIdAndFolloweeId`: 删除关注关系
  - `countFollowers`: 统计粉丝数
  - `countFollowees`: 统计关注数
  - `selectFolloweeIds`: 获取关注的用户ID列表

- **PostEeMapper.java** + **PostEeMapper.xml**:
  - `deleteByPostIdAndUserId`: 删除收藏关系
  - `countByPostId`: 统计收藏数

- **PostMapper.java** + **PostMapper.xml**:
  - `selectFollowFeed`: 查询关注用户的动态（已存在）
  - `selectCollectFeed`: 查询用户收藏的动态（新增）

#### 2.3 Service层
- **FollowService.java**:
  - `toggleFollow`: 关注/取消关注
  - `isFollowing`: 检查是否已关注
  - `getFollowerCount`: 获取粉丝数
  - `getFolloweeCount`: 获取关注数
  - `getFolloweeIds`: 获取关注的用户ID列表

- **PostService.java** (更新):
  - `getFeed`: 添加了对"follow"和"collect" Tab的支持
  - `toggleEe`: 更新为同时操作Redis和数据库，确保收藏数据持久化

#### 2.4 Controller层
- **FollowController.java**:
  - `POST /api/follows/{followeeId}`: 关注/取消关注
  - `GET /api/follows/check/{followeeId}`: 检查是否已关注
  - `GET /api/follows/list`: 获取关注列表

- **PostController.java** (已存在):
  - `GET /api/posts/feed`: 支持tab参数(all/follow/recommend/collect)

## 技术要点

### 1. 关注功能
- 使用唯一索引 `(follower_id, followee_id)` 防止重复关注
- 不能关注自己的业务逻辑校验
- 支持关注/取消关注的切换操作

### 2. 收藏功能优化
- 原有收藏功能只使用Redis存储，现在同时持久化到数据库
- Redis用于快速查询和缓存，数据库用于持久化和复杂查询
- 收藏Tab可以正确返回用户收藏的动态列表

### 3. 性能优化
- 使用JOIN查询优化关注动态和收藏动态的查询
- 按时间倒序排列，最新的动态优先展示
- 支持分页查询，避免一次性加载过多数据
