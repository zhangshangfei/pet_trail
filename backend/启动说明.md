# 后端启动说明

## 环境要求
- Java 21+
- MySQL 8.0+
- Redis 7.0+
- (可选) RabbitMQ 3.x

## 快速启动

### 1. 数据库初始化
```bash
# 创建数据库
mysql -u root -p -e "CREATE DATABASE pet_trail DEFAULT CHARSET utf8mb4;"

# 初始化基础表
mysql -u root -p pet_trail < backend/src/main/resources/init.sql

# 初始化新增表（第二阶段）
mysql -u root -p pet_trail < backend/src/main/resources/schema-v2.sql
```

### 2. 修改配置
编辑 `backend/src/main/resources/application.yml`：
```yaml
spring:
  datasource:
    username: root
    password: 你的 MySQL 密码  # 默认：root123654

  redis:
    host: localhost
    port: 6379

  rabbitmq:
    host: localhost
    port: 5672
    username: guest
    password: guest
```

### 3. 编译项目
```bash
cd backend
mvn clean package -DskipTests
```

### 4. 启动后端
```bash
java -jar target/pet-trail-backend-1.0.0.jar
```

或者使用 Maven：
```bash
mvn spring-boot:run
```

## 常见问题

### 问题 1：Maven 未安装
**解决方案**：
1. 下载 Maven：https://maven.apache.org/download.cgi
2. 解压并配置环境变量
3. 运行 `mvn -version` 验证

### 问题 2：Java 版本不对
**解决方案**：
- 确保使用 Java 21 或更高版本
- 运行 `java -version` 检查版本

### 问题 3：Redis 未启动
**解决方案**：
1. 下载 Redis：https://github.com/microsoftarchive/redis/releases
2. 启动 Redis 服务
3. 验证：`redis-cli ping` 应返回 `PONG`

### 问题 4：RabbitMQ 未安装（可选）
如果不想使用 RabbitMQ，可以：
1. 注释掉 `application.yml` 中的 `spring.rabbitmq` 配置
2. 在 `PostService.java` 中注释掉 `sendPostCreateMessage` 方法调用

## 验证启动成功

访问：http://localhost:8080/api/users/profile

如果返回 JSON 表示成功：
```json
{
  "success": false,
  "code": 401,
  "message": "用户未登录"
}
```
（401 是正常的，表示服务已启动但需要登录）

## 测试 API

### 1. 微信登录
```bash
curl -X POST http://localhost:8080/api/users/login \
  -H "Content-Type: application/json" \
  -d '{"code":"test-code"}'
```

### 2. 获取宠物列表（需要 Token）
```bash
curl http://localhost:8080/api/pets \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 3. 发布动态（需要 Token）
```bash
curl -X POST http://localhost:8080/api/posts \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content":"测试动态","petId":1}'
```

### 4. 打卡（需要 Token）
```bash
curl -X POST "http://localhost:8080/api/checkin?itemId=1" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 5. 记录步数（需要 Token）
```bash
curl -X POST "http://localhost:8080/api/health/steps?steps=1000" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
