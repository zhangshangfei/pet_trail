# 云托管请求诊断指南

## 问题现象

错误信息：`POST http://localhost:8080/api/users/login net::ERR_CONNECTION_REFUSED`

这说明请求仍然发到了本地地址，而不是使用云托管。

## 已修复的问题

### 原因分析

条件编译 `#ifdef MP-WEIXIN` 在某些情况下可能不会正确生效，导致：
- `BASE_URL` 被设置为 `http://localhost:8080`
- 请求判断逻辑错误地使用了 HTTP 请求

### 修复方案

1. **新增运行时环境检测** `isMpWeixin()` 函数
2. **使用配置项判断** 而不是仅依赖条件编译
3. **增加详细日志** 便于调试

### 修改的文件

- ✅ `frontend/src/config/env.js` - 新增 `IS_MP_WEIXIN` 配置
- ✅ `frontend/src/utils/request.js` - 使用新的判断逻辑
- ✅ `frontend/src/main.js` - 增加环境配置日志

## 验证步骤

### 步骤 1: 重新构建

```bash
cd frontend
npm run build:mp-weixin
```

✅ 已完成构建

### 步骤 2: 打开微信开发者工具

1. 打开微信开发者工具
2. 导入项目：`dist/build/mp-weixin/` 目录
3. **清除缓存**：工具栏 → 清缓存 → 全部清除
4. **重新编译**：点击编译按钮

### 步骤 3: 查看控制台日志

启动后应该看到以下日志：

```
✅ 微信云开发初始化完成
   环境ID: prod-3gyc3tpg28270da6
   云托管服务: springboot-4fyd
   已启用微信云托管模式

环境配置: {
  BASE_URL: "cloud",
  IS_MP_WEIXIN: true,
  USE_CLOUD: true,
  CLOUD_CONFIG: {...}
}
```

**关键点**：
- `BASE_URL` 应该是 `"cloud"`
- `IS_MP_WEIXIN` 应该是 `true`

### 步骤 4: 触发登录请求

点击登录按钮，查看控制台：

**正确的云托管请求日志**：
```
使用云托管请求: /api/users/login
云托管请求: {
  env: "prod-3gyc3tpg28270da6",
  service: "springboot-4fyd",
  path: "/api/users/login",
  method: "POST"
}
```

**错误的 HTTP 请求日志**（如果还是这样）：
```
使用 HTTP 请求: http://localhost:8080/api/users/login
```

## 仍然失败？排查清单

### ❌ 情况 1: BASE_URL 仍然是 http://localhost:8080

**原因**：条件编译未生效

**解决方案**：
1. 确认使用的是 `npm run build:mp-weixin` 构建
2. 确认导入的是 `dist/build/mp-weixin/` 目录
3. 清除缓存并重新编译

### ❌ 情况 2: IS_MP_WEIXIN 是 false

**原因**：运行环境检测失败

**解决方案**：
在微信开发者工具控制台执行：
```javascript
// 检查 wx 对象
console.log('wx:', wx)
console.log('wx.cloud:', wx.cloud)
```

如果 `wx` 不存在，说明不是在微信小程序环境运行。

### ❌ 情况 3: 云开发初始化失败

**症状**：没有看到 "✅ 微信云开发初始化完成" 日志

**解决方案**：
1. 确认 `project.config.json` 中的 `appid` 正确
2. 确认已在微信公众平台开通云开发
3. 确认环境 ID 正确

### ❌ 情况 4: 云托管请求失败

**症状**：看到 "云托管请求" 日志，但请求失败

**排查步骤**：

1. **检查云托管服务状态**
   - 登录微信公众平台
   - 进入云开发控制台
   - 查看云托管服务是否运行中

2. **检查服务名称**
   ```javascript
   // 应该是 'springboot-4fyd'
   console.log(CLOUD_CONFIG.service)
   ```

3. **检查环境 ID**
   ```javascript
   // 应该是 'prod-3gyc3tpg28270da6'
   console.log(CLOUD_CONFIG.env)
   ```

4. **查看云托管日志**
   - 云开发控制台 → 云托管 → 服务 → 日志查询
   - 查看是否有请求到达

## 临时解决方案

如果云托管暂时无法使用，可以临时改回 HTTP 请求：

### 方法 1: 修改环境配置

编辑 `frontend/src/config/env.js`：

```javascript
const getApiBaseUrl = () => {
  // 临时改为本地开发
  return 'http://localhost:8080'
}
```

然后确保本地后端服务已启动：
```bash
cd backend
docker-compose up -d
mvn spring-boot:run
```

### 方法 2: 使用 H5 模式调试

```bash
cd frontend
npm run dev
```

这会在浏览器中打开 H5 版本，使用 HTTP 请求。
