**修改方案：**
```java

    followMapper.insert(follow);
    return true;
} catch (DuplicateKeyException e) {

}
```

### 5. 优化关注列表API
**问题：** 当前只返回用户ID列表，前端需要额外请求才能展示用户信息

**需要实现：**
- [ ] 修改 `/api/follows/list` 返回包含用户详细信息的列表
- [ ] 返回字段：userId, nickname, avatar, petInfo, followerCount 等

**修改方案：**
```java
@GetMapping("/list")
public Result<List<UserVO>> getFollowList() {
    Long followerId = UserContext.getCurrentUserId();
    List<Long> followeeIds = followService.getFolloweeIds(followerId);
    List<UserVO> users = followeeIds.stream()
        .map(id -> userService.getUserVO(id))
        .collect(Collectors.toList());
    return Result.success(users);
}
```

**需要实现：**
- [ ] 在 `selectFollowFeed` 查询中 JOIN users 表，过滤掉已删除的用户
- [ ] 在 `users` 表中添加 `status` 字段（1-正常，0-已删除）

**SQL 修改：**
```sql
SELECT p.* FROM posts p
INNER JOIN follows f ON p.user_id = f.followee_id
INNER JOIN users u ON p.user_id = u.id
WHERE f.follower_id = #{followerId}
AND p.status = 1
AND u.status = 1
ORDER BY p.created_at DESC
LIMIT #{offset}, #{limit}
```
