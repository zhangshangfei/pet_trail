# 微信云托管指南

> 最后更新：2026年4月20日

## 一、概述

本项目前端已配置微信云托管，微信小程序环境下自动使用云托管服务调用后端 API，H5 环境继续使用普通 HTTP 请求，并支持降级策略。

### 云托管环境

| 配置项 | 值 |
|--------|-----|
| 环境 ID | `prod-3gyc3tpg28270da6` |
| 服务名称 | `springboot-4fyd` |
| AppID | `wxb210d75568abe1f7` |

### 环境对照

| 环境 | API 基础地址 | 请求方式 |
|------|-------------|---------|
| 微信小程序 | `cloud` (标记) | `wx.cloud.callContainer()` |
| H5 开发 | `http://localhost:8080` | `uni.request()` |
| H5 生产 | `https://api.pettrail.com` | `uni.request()` |

## 二、代码改动说明

### 1. 环境配置 (`src/config/env.js`)

- 新增 `USE_CLOUD: true` 配置项，用于运行时判断是否使用云托管

```javascript
{
  VITE_API_BASE_URL: 'cloud',
  VITE_CLOUD_CONFIG: {
    env: 'prod-3gyc3tpg28270da6',
    service: 'springboot-4fyd'
  },
  USE_CLOUD: true
}
```

- 新增 `ensureCloudInit()` 函数，检查云开发是否初始化

- 优化请求拦截器 `request()` 函数：使用 `USE_CLOUD && wx.cloud` 判断

### 3. 应用入口 (`src/main.js`)

- 完善云开发初始化日志
- 增加云托管服务名称输出和模式启用提示

### 4. 小程序配置 (`src/manifest.json`)

- 新增 `cloudfunctionRoot` 配置
- 新增 `permission` 和 `requiredPrivateInfos` 配置

## 三、请求流程

### 微信小程序环境

```
用户操作 → 前端 API 调用 → request() 函数
  → 检查 USE_CLOUD && wx.cloud (true)
  → cloudRequest() → wx.cloud.callContainer()
  → 云托管服务 (springboot-4fyd) → Spring Boot 后端 → 返回响应
```

### H5 环境

```
用户操作 → 前端 API 调用 → request() 函数
  → 检查环境 (非微信小程序)
  → httpRequest() → uni.request()
  → HTTP 请求 (localhost:8080 或 api.pettrail.com) → 返回响应
```

### 云托管请求格式

```javascript
wx.cloud.callContainer({
  config: { env: 'prod-3gyc3tpg28270da6' },
  path: '/api/users/login',
  header: {
    'X-WX-SERVICE': 'springboot-4fyd',
    'Content-Type': 'application/json',
    'Authorization': 'Bearer <token>'
  },
  method: 'POST',
  data: { code: 'xxx' }
})
```

> **关键点**：`X-WX-SERVICE` 请求头必须设置，否则云托管无法路由请求。

### 降级策略

当云开发不可用时，自动降级为普通 HTTP 请求：

```javascript
if (!wx.cloud) {
  console.error('当前环境不支持微信云开发，降级为普通 HTTP 请求')
  return httpRequest(options)
}
```

## 四、部署步骤

### 前置条件

1. 已注册微信小程序账号
2. 已开通微信云开发服务
3. 已创建云托管环境（环境 ID: `prod-3gyc3tpg28270da6`）
4. 已安装微信开发者工具
5. 已安装 Docker（用于构建镜像）

### 步骤 1: 构建后端 Docker 镜像

```bash
docker build -t pet-trail-backend:latest .
```

### 步骤 2: 推送镜像到云托管

**方式 A: 使用微信开发者工具**

1. 打开微信开发者工具 → "云开发" → "云托管"
2. 选择服务 `springboot-4fyd` → "版本管理" → "新建版本"
3. 上传 Docker 镜像或使用 Dockerfile 构建

**方式 B: 使用命令行工具**

```bash
cloudbase login
cloudbase cloudrun push-image \
  --env-id prod-3gyc3tpg28270da6 \
  --service-name springboot-4fyd \
  --image pet-trail-backend:latest
```

### 步骤 3: 部署服务

1. 在云托管控制台选择刚推送的镜像
2. 配置环境变量（参考下方环境变量表）
3. 配置端口（默认 8080）
4. 点击"部署"

### 步骤 4: 构建并上传小程序

```bash
cd frontend
npm install
npm run build:mp-weixin
```

然后在微信开发者工具中打开 `dist/build/mp-weixin/` 目录，上传代码到微信公众平台。

## 五、环境变量配置

### 后端 application.yml

```yaml
spring:
  datasource:
    url: ${DB_URL:jdbc:mysql://localhost:3306/pet_trail?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai}
    username: ${DB_USERNAME:pettrail}
    password: ${DB_PASSWORD:pettrail123}
  redis:
    host: ${REDIS_HOST:localhost}
    port: ${REDIS_PORT:6379}
```

### 云托管控制台环境变量

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `DB_URL` | 数据库连接字符串 | `jdbc:mysql://mysql:3306/pet_trail...` |
| `DB_USERNAME` | 数据库用户名 | `pettrail` |
| `DB_PASSWORD` | 数据库密码 | `pettrail123` |
| `REDIS_HOST` | Redis 主机地址 | `redis` |
| `REDIS_PORT` | Redis 端口 | `6379` |
| `ALIYUN_OSS_ACCESS_KEY_ID` | 阿里云 OSS AccessKey ID | `LTAI5t...` |
| `ALIYUN_OSS_ACCESS_KEY_SECRET` | 阿里云 OSS AccessKey Secret | `xxx` |
| `JWT_SECRET` | JWT 密钥 | `your-secret-key` |

## 六、本地开发

### 前端开发

```bash
cd frontend
npm run dev              # H5 开发模式
npm run dev:mp-weixin    # 微信小程序开发模式
```

> 微信小程序开发模式下，前端会尝试调用云托管服务。如果云托管服务未部署，可临时修改 `frontend/src/config/env.js` 将 API 地址改为 `http://localhost:8080`。

### 后端开发

```bash
cd backend
docker-compose up -d mysql redis   # 启动依赖服务
mvn spring-boot:run                # 运行后端服务
```

## 八、故障排查

| 问题 | 症状 | 解决方案 |
|------|------|---------|
| 云托管请求失败 | 控制台显示"云托管请求失败" | 检查云托管服务状态、环境ID、服务名称、网络连接 |
| Token 验证失败 | 返回 401 错误 | 检查 JWT_SECRET 环境变量、Token 过期时间、清除缓存重新登录 |
| 数据库连接失败 | 后端启动失败 | 检查数据库环境变量、数据库服务可访问性、用户名密码 |
| 文件上传失败 | 上传图片失败 | 检查阿里云 OSS 配置、AccessKey 权限、文件大小限制(10MB) |
| 云开发未初始化 | 控制台显示"不支持微信云开发" | 确认在微信小程序环境下运行、`wx.cloud.init()` 已调用、`project.config.json` 配置正确 |

## 九、监控与优化

### 监控告警

建议在云托管控制台配置告警：
- 错误率 > 5% 时告警
- 响应时间 > 3s 时告警
- 实例数达到上限时告警

### 性能优化

- 数据库连接池：`hikari.maximum-pool-size: 20`
- Redis 缓存：用户信息 1h、宠物信息 1h、动态列表 10min
- 图片优化：上传前压缩 + CDN 加速

### 安全建议

- 不要在代码中硬编码敏感信息
- 使用云托管环境变量管理密钥
- 所有 API 需要 JWT Token 认证
- 定期更换密钥
