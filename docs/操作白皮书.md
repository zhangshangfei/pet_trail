## 二、部署与运维

### 2.1 环境要求

| 组件 | 最低要求 |
|------|----------|
| 服务器 | 2 核 4G（推荐 4 核 8G） |
| 操作系统 | Linux (CentOS 7+/Ubuntu 20.04+) |
| Docker | 20.10+ |
| Docker Compose | 2.0+ |
| 域名 | 已备案域名 + SSL 证书 |
| 微信小程序 | 已注册小程序账号 |

### 2.2 Docker Compose 服务编排

共 5 个服务，通过 `pet-trail-network` 桥接网络互联：

| 服务 | 镜像 | 容器名 | 端口 | 依赖 |
|------|------|--------|------|------|
| mysql | mysql:8.0 | pet-trail-mysql | 3306:3306 | 无 |
| redis | redis:7-alpine | pet-trail-redis | 6379:6379 | 无 |
| rabbitmq | 自定义(含延迟插件) | pet-trail-rabbitmq | 5672:5672, 15672:15672 | 无 |
| backend | 自定义 | pet-trail-backend | 8080:8080 | mysql, redis |
| nginx | nginx:alpine | pet-trail-nginx | 80:80, 443:443 | backend |

### 2.3 环境变量配置

后端服务通过 20+ 个环境变量注入敏感配置：

| 变量分类 | 变量示例 | 说明 |
|----------|----------|------|
| MySQL | MYSQL_HOST, MYSQL_PORT, MYSQL_DATABASE, MYSQL_USER, MYSQL_PASSWORD | 数据库连接 |
| Redis | REDIS_HOST, REDIS_PORT, REDIS_PASSWORD | Redis 连接 |
| JWT | JWT_SECRET | Token 密钥 |
| 腾讯云 COS | COS_SECRET_ID, COS_SECRET_KEY, COS_BUCKET_NAME, COS_REGION | 对象存储 |
| 微信 | WX_APPID, WX_SECRET, WX_MCH_ID, WX_API_KEY | 小程序 + 支付 |
| AI | AI_OPENROUTER_API_KEY, AI_ZHIPU_API_KEY | AI 模型密钥 |
| RabbitMQ | RABBITMQ_ENABLED, RABBITMQ_HOST, RABBITMQ_USER, RABBITMQ_PASSWORD | 消息队列 |

### 2.4 部署步骤

```bash
# 1. 克隆项目
git clone <repo-url> && cd pet-trail

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 填入实际配置

# 3. 构建管理后台
cd admin && pnpm install && pnpm build && cd ..

# 4. 启动所有服务
docker-compose up -d

# 5. 查看服务状态
docker-compose ps

# 6. 查看后端日志
docker-compose logs -f backend
```

### 2.5 Nginx 关键配置

```nginx
# 文件上传限制
client_max_body_size 50m;

# 大文件上传超时
proxy_send_timeout 180s;
proxy_read_timeout 180s;

# 后端反向代理
location /api/ {
    proxy_pass http://backend:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}

# 管理后台静态资源
location /admin/ {
    alias /usr/share/nginx/html/admin/;
    try_files $uri $uri/ /admin/index.html;
}
```

### 2.6 常用运维命令

```bash
# 重启单个服务
docker-compose restart backend

# 查看日志
docker-compose logs -f --tail=100 backend

# 进入容器
docker-compose exec backend bash
docker-compose exec mysql mysql -u root -p pet_trail

# 清理并重建
docker-compose down -v && docker-compose up -d --build

# 数据库备份
docker-compose exec mysql mysqldump -u root -p pet_trail > backup.sql

# Redis 缓存清理
docker-compose exec redis redis-cli -a <password> FLUSHDB
```

## 三、用户端操作指南

### 3.1 登录与注册

#### 操作流程

1. 打开微信小程序，自动检测登录状态
2. 未登录时，点击需登录的功能会弹出登录提示
3. 点击"去登录"→ 微信授权 → 自动完成登录
4. 首次登录自动创建账号，生成默认昵称

#### 注意事项

- Token 有效期 7 天，过期自动弹窗重新登录
- 同一微信 code 只能使用一次（防重放攻击）
- 退出登录后 Token 加入黑名单立即失效

### 3.2 宠物管理

#### 添加宠物

1. 进入"我的"页面 → 点击宠物列表旁的"+"按钮
2. 填写宠物信息：
   - **名称**（必填）
   - **类别**：猫 / 狗 / 其他
   - **品种**（如英短、金毛等）
   - **性别**：未知 / 公 / 母
   - **生日**
   - **体重**（kg）
   - **毛色**
   - **绝育状态**：未绝育 / 已绝育
3. 可选上传宠物头像
4. 点击保存

#### 管理宠物

- **切换宠物**：在首页/健康看板/打卡页顶部点击宠物头像切换
- **编辑宠物**：宠物详情页 → 编辑按钮
- **删除宠物**：宠物详情页 → 删除按钮（二次确认）
- **成长相册**：宠物详情页 → 成长相册 → 上传/删除照片
- **宠物日历**：宠物详情页 → 日历视图查看事件

### 3.3 健康管理

#### 健康看板

1. 点击底部 TabBar"健康看板"
2. 选择宠物 → 展示：
   - 体重趋势图（7天/30天切换）
   - 待办疫苗提醒（7天内紧急标记）
   - 待办驱虫提醒（7天内紧急标记）
   - AI 健康分析入口

#### 疫苗提醒

1. 健康看板 → 点击"疫苗" → 进入疫苗列表
2. 点击"+"添加疫苗提醒：
   - 疫苗名称（如狂犬疫苗、猫三联等）
   - 疫苗类型：核心 / 非核心
   - 下次接种日期
   - 提前提醒天数
3. 疫苗到期时自动收到站内信 + 微信订阅消息
4. 接种完成后点击"标记完成"→ 触发微信订阅消息授权

#### 驱虫提醒

1. 健康看板 → 点击"驱虫" → 进入驱虫列表
2. 点击"+"添加驱虫提醒：
   - 驱虫类型：体内 / 体外
   - 产品名称
   - 下次驱虫日期
   - 间隔天数
3. 驱虫到期时自动收到站内信 + 微信订阅消息
4. 完成后点击"标记完成"

#### 体重记录

1. 健康看板 → 点击"体重" → 进入体重记录
2. 点击"+"添加体重记录：
   - 体重值（kg）
   - 记录日期
   - 备注（可选）
3. 体重趋势图展示历史变化

#### AI 健康分析

1. 健康看板 → 点击"AI 健康分析"
2. 系统调用 AI 模型分析宠物健康数据
3. 展示综合评分、健康等级、警告建议
4. 支持切换 AI 模型（DeepSeek / GLM-4-Flash / 通用免费）

### 3.4 打卡系统

#### 每日打卡

1. 点击底部 TabBar"每日打卡"
2. 选择宠物 → 展示打卡网格
3. 点击未打卡项 → 震动反馈 + 成功动画
4. 可批量打卡多个项目

#### 打卡项管理

- **系统默认**：16 个预置打卡项（喂食/遛狗/喝水/洗澡/梳毛/训练/刷牙/吃药/量体温/称体重/剪指甲/清猫砂/社交/散步/玩耍/拍照）
- **添加自定义**：打卡页 → 工具栏"添加"→ 填写名称/图标/类型
- **管理打卡项**：打卡页 → 工具栏"管理"→ 编辑/删除/隐藏/显示
- **打卡提醒**：打卡页 → 工具栏"提醒"→ 设置提醒时间

#### 打卡记录

- 默认显示最近 7 天记录
- 点击"查看全部"展开所有记录（按月分组）
- 点击"加载更多"加载更早月份（最多回溯 2 年）
- 打卡报表：周报/月报，含完成率统计

### 3.5 社区动态

#### 浏览动态

1. 点击底部 TabBar"首页"
2. 切换标签页：
   - **全部**：所有动态
   - **收藏**：已收藏的动态
   - **关注**：关注用户的动态
   - **推荐**：算法推荐的动态
3. 下拉刷新 / 上拉加载更多

#### 发布动态

1. 点击首页右下角悬浮"+"按钮
2. 填写内容：
   - 文字内容
   - 图片（最多 9 张，自动压缩至 5MB 以下）
   - 视频（不压缩，上传超时 180 秒）
   - 关联宠物（可选）
   - 话题标签（可选）
   - 关联挑战赛（可选）
   - 位置信息（可选）
3. 点击发布 → 自动进入内容审核队列

#### 互动操作

- **点赞**：点击爱心图标
- **收藏**：点击收藏图标
- **评论**：点击评论图标 → 输入内容 → 支持回复他人评论
- **分享**：点击分享图标
- **删除**：自己的动态/评论可删除
- **举报**：动态详情页/用户详情页的举报入口

### 3.6 通知消息

1. 点击首页右上角铃铛图标 → 进入通知列表
2. 按类型筛选：全部/点赞/评论/关注/系统
3. 点击通知 → 标记已读 + 跳转对应页面
4. 长按通知 → 删除
5. 支持全部已读 / 清空所有

### 3.7 其他功能

#### 挑战赛

1. 首页 → "我的挑战"入口 → 查看已参与的挑战
2. 挑战列表 → 查看进行中的挑战 → 参加挑战
3. 挑战详情 → 查看条件/奖励/排行榜

#### 宠物商城

1. 首页 → "商城"入口
2. 分类浏览：推荐/粮食/玩具/保健/用品
3. 点击商品 → 查看详情（价格/品牌/销量/评分/来源平台）

#### 宠物医院

1. 首页 → "宠物医院"入口
2. 浏览医院列表 → 查看详情（地址/电话/营业时间/评分/专科）
3. 在线预约：选择宠物 + 日期 + 时间 + 症状
4. 我的预约列表 → 查看预约状态/取消预约

#### 成就墙

1. "我的"页面 → "成就墙"入口
2. 查看成就列表（按类型筛选：打卡/健康/社交/成长）
3. 已解锁成就可领取奖励
4. 成就自动检测解锁（每日 2:00 定时检查）

#### 会员系统

1. "我的"页面 → "宠迹 Pro"入口
2. 选择订阅计划（月度/年度）
3. 微信支付
4. 查看会员信息/取消订阅

#### 意见反馈

1. "我的"页面 → "意见反馈"入口
2. 选择类型：Bug / 功能建议 / 体验优化 / 其他
3. 填写内容 + 上传图片（可选）+ 联系方式（可选）
4. 提交后跳转反馈记录列表

## 四、管理后台操作指南

### 4.1 登录

1. 访问 `https://域名/admin/`
2. 输入用户名 + 密码
3. 如已绑定 2FA，输入 TOTP 验证码
4. 登录成功 → 跳转仪表盘

### 4.2 仪表盘

仪表盘提供运营数据总览：

| 区域 | 内容 |
|------|------|
| 统计卡片 | 总用户数、今日活跃、总动态数、总举报数 |
| 趋势图 | 支持 7/30/90 天切换，展示新增用户/活跃用户/新增动态 |
| 审核饼图 | 待审核/已通过/已拒绝比例 |
| 今日数据 | 新增用户、新增动态、活跃用户 |
| 待处理事项 | 待审核动态数、待处理举报数 |

### 4.3 用户管理

#### 操作步骤

1. 左侧菜单 → 用户管理
2. 搜索筛选：关键词（昵称/OpenID）、状态（正常/禁用）
3. 查看用户详情：基本信息 + 统计数据（宠物数/动态数/获赞数/粉丝数/关注数）
4. 启用/禁用用户（需 SUPER_ADMIN 权限）
5. 导出用户数据（需 `export` 按钮权限）

### 4.4 动态管理

#### 审核动态

1. 左侧菜单 → 动态管理
2. 筛选审核状态：待审核/已通过/已拒绝
3. **单条审核**：点击"通过"或"拒绝"（可填备注）
4. **批量审核**：勾选多条 → 批量通过/拒绝
5. 审核结果通过站内信通知用户

#### 其他操作

- **删除动态**：逻辑删除，进入回收站
- **回收站**：查看已删除动态，支持恢复
- **导出**：导出动态数据为 Excel

### 4.5 评论管理

1. 左侧菜单 → 评论管理
2. 搜索筛选：关键词、动态 ID、状态
3. 删除/恢复评论

### 4.6 举报管理

#### 处理举报

1. 左侧菜单 → 举报管理
2. 筛选：处理状态（待处理/已处理/已驳回）、举报类型（动态/评论/用户）
3. 点击"处理"→ 填写处理备注
4. **联动处罚**（可选）：
   - 下架内容：自动将动态设为删除状态
   - 禁言用户：自动将用户设为禁用状态
5. 驳回举报：填写驳回备注

### 4.7 反馈管理

1. 左侧菜单 → 反馈管理
2. 筛选：状态（待处理/处理中/已回复）、类型
3. 回复反馈：
   - 填写回复内容
   - 选择状态：处理中 / 已回复
4. 删除反馈

### 4.8 通知管理

1. 左侧菜单 → 通知管理
2. 查看通知列表（类型/阅读状态筛选）
3. **发送通知**：输入用户 ID + 标题 + 内容 → 发送给指定用户
4. **广播通知**：输入标题 + 内容 → 发送给所有用户（需 SUPER_ADMIN 权限）

### 4.9 宠物管理

1. 左侧菜单 → 宠物管理
2. 搜索筛选：名称、类别
3. 查看详情（含主人信息）
4. 编辑宠物信息 / 删除宠物
5. 导出宠物数据

### 4.10 挑战赛管理

1. 左侧菜单 → 挑战赛配置
2. 新增挑战赛：
   - 标题 + 描述 + 封面图
   - 类型：打卡/社交/健康
   - 条件类型（8 种）+ 目标值
   - 奖励描述
   - 开始/结束时间
3. 上线/下线挑战赛
4. 查看统计：参与人数/完成率/参与者列表
5. 导出挑战赛数据

### 4.11 商城管理

1. 左侧菜单 → 商城管理
2. 新增商品：
   - 名称 + 描述 + 封面图
   - 分类：粮食/玩具/保健/用品
   - 品牌 + 售价 + 原价 + 佣金比例

## 七、消息队列

### 7.1 RabbitMQ 架构

```
┌──────────────────────────────────────────────────────┐
│  reminder.delayed.exchange (延迟交换机)               │
│  类型: x-delayed-message                              │
├──────────────┬───────────────────────────────────────┤
│  routing key │  目标队列                              │
├──────────────┼───────────────────────────────────────┤
│  feeding     │  reminder.feeding.queue (带死信路由)   │
│  checkin     │  reminder.checkin.queue (带死信路由)   │
└──────────────┴───────────────────────────────────────┘

┌──────────────────────────────────────────────────────┐
│  reminder.dlx.exchange (死信交换机)                   │
├──────────────┬───────────────────────────────────────┤
│  routing key │  目标队列                              │
├──────────────┼───────────────────────────────────────┤
│  feeding.dlq │  reminder.feeding.dlq (喂食死信队列)   │
│  checkin.dlq │  reminder.checkin.dlq (打卡死信队列)   │
└──────────────┴───────────────────────────────────────┘
```

### 7.2 消息投递流程

1. 用户创建喂食/打卡提醒 → 生产者发送延迟消息到 RabbitMQ
2. 延迟时间到达 → 消息投递到目标队列
3. 消费者消费消息：
   - 校验提醒有效性（是否仍启用）
   - 判断是否今日需发送
   - 发送站内信 + 微信订阅消息
   - 调度下次提醒（循环投递）
4. 消费失败 → 消息进入死信队列

### 7.3 降级策略

- RabbitMQ 未启用（`RABBITMQ_ENABLED=false`）时，自动降级到 `FallbackReminderTask` 定时轮询
- 定时轮询每分钟检查一次，精度低于 MQ 延迟消息

## 八、缓存架构

### 8.1 多级缓存

```
请求 → Caffeine (L1 本地缓存)
         ↓ 未命中
       Redis (L2 分布式缓存)
         ↓ 未命中
       MySQL (数据库)
         ↓ 查询结果
       回写 Redis + Caffeine
```

### 8.2 缓存配置

|--------|--------|----------|------|

| posts | Caffeine + Redis | 10min | 动态信息 |
| general | Caffeine + Redis | 1h | 通用缓存 |
| config | Redis | 24h | 系统配置 |

### 8.3 缓存一致性

- 数据更新时自动清除对应缓存
- 管理后台支持手动刷新配置缓存
- AI 模型缓存支持按宠物清空

## 十、故障排查

### 10.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 小程序请求超时 | 网络波动 / 超时配置过短 | 检查 `env.js` 中 `REQUEST_TIMEOUT` 配置（默认 30s） |
| 图片上传 413 错误 | Nginx 上传限制 | 检查 `nginx.conf` 中 `client_max_body_size`（需 50m） |
| iOS 日期显示 NaN | iOS 不支持 `-` 分隔符 | 确认代码中使用 `.replace(/-/g, '/')` 转换 |
| Live Photo 上传失败 | iOS Live Photo 格式 | 确认使用 `uni.chooseMedia` + `sizeType: ['compressed']` |
| 订阅消息授权失败 | 非用户手势触发 | 确认 `requestSubscribeMessage` 在用户点击事件中调用 |
| 管理员删除失败 | MyBatis-Plus 逻辑删除 | 使用 `deleteById()` 而非 `updateById()` 设置 deleted 字段 |
| AI 分析超时 | 模型响应慢 | 检查 AI 模型配置 / 切换到更快的模型 |
| 打卡提醒未发送 | RabbitMQ 未启用 | 检查 `RABBITMQ_ENABLED` 环境变量 |
