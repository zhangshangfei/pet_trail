# 微信云托管快速启动指南

## 前置条件

1. 已注册微信小程序账号
2. 已开通微信云开发服务
3. 已创建云托管环境（环境 ID: `prod-3gyc3tpg28270da6`）
4. 已安装微信开发者工具
5. 已安装 Docker（用于构建镜像）

## 环境配置

### 1. 前端配置

前端已配置云托管调用，配置文件位于：
- `frontend/src/config/env.js` - 云托管环境配置
- `frontend/src/utils/request.js` - 云托管请求封装
- `frontend/src/main.js` - 云开发初始化

**关键配置项**：

```javascript
// 云托管环境 ID
env: 'prod-3gyc3tpg28270da6'

// 云托管服务名称
service: 'springboot-4fyd'
```

### 2. 后端配置

后端需要适配微信云托管环境，主要配置：

#### application.yml

```yaml
spring:
  datasource:
    url: ${DB_URL:jdbc:mysql://localhost:3306/pet_trail?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai}
    username: ${DB_USERNAME:pettrail}
    password: ${DB_PASSWORD:pettrail123}
  redis:
    host: ${REDIS_HOST:localhost}
    port: ${REDIS_PORT:6379}

# 云托管环境配置
cloud:
  env: ${CLOUD_ENV:prod-3gyc3tpg28270da6}
  service: ${CLOUD_SERVICE:springboot-4fyd}
```

#### 环境变量

在云托管控制台配置以下环境变量：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `DB_URL` | 数据库连接字符串 | `jdbc:mysql://mysql:3306/pet_trail...` |
| `DB_USERNAME` | 数据库用户名 | `pettrail` |
| `DB_PASSWORD` | 数据库密码 | `pettrail123` |
| `REDIS_HOST` | Redis 主机地址 | `redis` |
| `REDIS_PORT` | Redis 端口 | `6379` |
| `ALIYUN_OSS_ACCESS_KEY_ID` | 阿里云 OSS AccessKey ID | `LTAI5t...` |
| `ALIYUN_OSS_ACCESS_KEY_SECRET` | 阿里云 OSS AccessKey Secret | `xxx` |
| `JWT_SECRET` | JWT 密钥 | `your-secret-key` |

## 部署步骤

### 步骤 1: 构建后端 Docker 镜像

```bash
# 在项目根目录执行
docker build -t pet-trail-backend:latest .
```

### 步骤 2: 推送镜像到云托管

有两种方式：

#### 方式 A: 使用微信开发者工具推送

1. 打开微信开发者工具
2. 导入前端项目 `frontend/` 目录
3. 点击"云开发"按钮
4. 进入"云托管"控制台
5. 点击"服务管理" -> "springboot-4fyd"
6. 点击"版本管理" -> "新建版本"
7. 上传 Docker 镜像或使用 Dockerfile 构建

#### 方式 B: 使用命令行工具

```bash
# 登录微信云托管 CLI
cloudbase login

# 推送镜像
cloudbase cloudrun push-image \
  --env-id prod-3gyc3tpg28270da6 \
  --service-name springboot-4fyd \
  --image pet-trail-backend:latest
```

### 步骤 3: 部署服务

1. 在云托管控制台选择刚推送的镜像
2. 配置环境变量（参考上面的环境变量表）
3. 配置端口（默认 8080）
4. 点击"部署"

### 步骤 4: 构建并上传小程序

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 构建微信小程序
npm run build:mp-weixin
```

然后在微信开发者工具中：

1. 打开 `dist/build/mp-weixin/` 目录
2. 点击"上传"按钮
3. 填写版本号和备注
4. 提交审核

## 本地开发

### 前端开发

```bash
cd frontend

# 开发模式（H5）
npm run dev

# 开发模式（微信小程序）
npm run dev:mp-weixin
```

**注意**：微信小程序开发模式下，前端会尝试调用云托管服务。如果云托管服务未部署，可以临时修改 `frontend/src/config/env.js`：

```javascript
const getApiBaseUrl = () => {
  // #ifdef MP-WEIXIN
  // 临时改为本地开发
  return 'http://localhost:8080'
  // #endif
  // ...
}
```

### 后端开发

```bash
cd backend

# 使用 Docker Compose 启动依赖服务
docker-compose up -d mysql redis

# 运行后端服务
mvn spring-boot:run
```

## 常见问题

### Q1: 云托管请求失败

**问题**: 控制台显示 "云托管请求失败"

**解决方案**:
1. 检查云托管服务是否正常运行
2. 检查环境 ID 和服务名称是否正确
3. 查看云托管日志排查错误
4. 检查网络连接是否正常

### Q2: Token 验证失败

**问题**: 返回 401 错误

**解决方案**:
1. 检查 JWT_SECRET 环境变量是否正确配置
2. 检查后端 Token 验证逻辑
3. 清除本地缓存重新登录

### Q3: 数据库连接失败

**问题**: 后端启动失败或报数据库连接错误

**解决方案**:
1. 检查数据库环境变量配置
2. 确保数据库服务可访问
3. 检查数据库用户名密码是否正确
4. 查看后端启动日志

### Q4: 文件上传失败

**问题**: 上传图片或文件失败

**解决方案**:
1. 检查阿里云 OSS 配置是否正确
2. 确认 AccessKey 有足够权限
3. 检查文件大小是否超过限制（10MB）
4. 查看后端日志

### 云托管监控

在云托管控制台可以查看：
- 请求量趋势
- 响应时间
- 错误率

2. 按关键词搜索

- 错误率 > 5% 时告警

- 实例数达到上限时告警

## 性能优化

### 1. 数据库连接池

确保后端配置了合理的数据库连接池：

```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000
```

### 2. Redis 缓存

充分利用 Redis 缓存热点数据：
- 用户信息缓存 1 小时
- 宠物信息缓存 1 小时
- 动态列表缓存 10 分钟

### 3. 图片优化

- 上传前压缩图片
- 使用阿里云 CDN 加速
- 配置合理的缓存策略

## 安全建议

### 1. 环境变量安全

- 不要在代码中硬编码敏感信息
- 使用云托管的环境变量管理功能
- 定期更换密钥

### 2. API 安全

- 所有 API 都需要 JWT Token 认证
- 实施请求限流
- 记录访问日志

### 3. 数据安全

- 数据库使用强密码
- 定期备份数据库
- 启用 SSL 连接
