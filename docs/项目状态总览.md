# 宠迹项目 - 综合分析文档

> 最后更新：2026年4月27日 | 版本：v4.1.0

## 一、项目概述

### 1.1 项目信息

- **项目名称**：宠迹 (Pet Trail)
- **产品定位**：宠物生活记录工具 + 轻社交社区
- **目标用户**：20-35岁宠物主人，女性偏多，有记录和分享习惯
- **核心价值**：科学记录宠物健康、养成良好养宠习惯、通过内容分享获得情感满足、建立宠物爱好者轻社交平台
- **官网**：[https://pettrail.openclaw.com](https://pettrail.openclaw.com)

### 1.2 项目地址

- **代码仓库**：[https://github.com/zhangshangfei/pet_trail](https://github.com/zhangshangfei/pet_trail)
- **问题反馈**：<https://github.com/zhangshangfei/pet_trail/issues>

#### 9.5.1 行为记录模块接入详细规划

**模块现状**：后端基础架构已完成（Entity/DTO/Mapper/Service/Controller），前端API已封装（`behavior.js`），但前端页面完全未接入，导致推荐算法行为维度始终返回0分。

**已完成部分**：

| 层级 | 文件 | 状态 |
|------|------|------|
| 数据库 | `user_behaviors` 表（7种行为类型） | ✅ 已创建 |
| Entity | `UserBehavior.java` | ✅ 已完成 |
| DTO | `UserBehaviorDTO.java`、`UserBehaviorViewDTO.java` | ✅ 已完成 |
| Mapper | `UserBehaviorMapper.java`（3个自定义查询） | ✅ 已完成 |
| Service | `UserBehaviorService.java`（7个记录方法+6个查询方法，@Async异步） | ✅ 已完成 |
| Controller | `UserBehaviorController.java`（2个API端点） | ✅ 已完成 |
| 前端API | `behavior.js`（recordBehavior/recordView） | ✅ 已封装 |
| 推荐算法集成 | `RecommendService.computeBehaviorScore()`（权重5%） | ✅ 已集成，但无数据流入 |

**待实施工作**：

**① 前端行为埋点接入（核心）**

| 场景 | 行为类型 | 调用方法 | 当前状态 |
|------|---------|---------|---------|
| 浏览动态卡片 | view | `recordView({ targetType: 'post', targetId, duration })` | 方法已定义但未触发 |
| 点赞动态 | like | `recordBehavior({ action: 'like', targetType: 'post', targetId })` | 未接入 |
| 收藏动态 | collect | `recordBehavior({ action: 'collect', targetType: 'post', targetId })` | 未接入 |
| 评论动态 | comment | `recordBehavior({ action: 'comment', targetType: 'post', targetId })` | 未接入 |
| 分享动态 | share | `recordBehavior({ action: 'share', targetType: 'post', targetId })` | 未接入 |
| 关注用户 | follow | `recordBehavior({ action: 'follow', targetType: 'user', targetId })` | 未接入 |
| 发布动态 | publish | 后端已自动记录 | ✅ 已完成 |

**② 浏览时长追踪**

- 动态卡片进入可视区域时记录开始时间（`IntersectionObserver`）
- 离开可视区域时计算停留时长并上报
- 页面卸载时（`onHide`/`onUnload`）补报未完成的浏览记录

**③ 后端补齐缺失调用**

| 行为 | 调用位置 | 当前状态 |
|------|---------|---------|
| comment | CommentController 评论时调用 `recordComment` | ❌ 未调用 |
| share | PostInteractionController 分享时调用 `recordShare` | ❌ 未调用 |
| follow | FollowController 关注时调用 `recordFollow` | ❌ 未调用 |

**④ 推荐算法权重调整**

当前 `PW_BEHAVIOR = 0.05`（5%），行为数据接入后可提升至 0.10-0.15，使行为维度对推荐结果产生实质性影响。

**⑤ 数据清理策略**

`user_behaviors` 表会持续增长，需设置：
- 定期归档历史数据（保留近90天明细）
- 聚合历史行为统计（月度行为频次汇总表）
- 考虑分表或分区（按月分区）
