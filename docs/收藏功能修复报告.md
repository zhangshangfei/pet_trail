## 修复详情

### ✅ 严重问题修复（3个）

#### 1. 修复 `isUserEeLiked` 方法 - 增加数据库回查
**问题：** Redis 数据丢失时，即使用户已收藏也会返回 false

**修复方案：**
- 在 `PostEeMapper` 中添加 `selectByPostIdAndUserId` 方法
- 在 `PostEeMapper.xml` 中添加对应的 SQL 查询
- 修改 `isUserEeLiked` 方法，Redis 未命中时回查数据库

**修改文件：**
- `backend/src/main/java/com/pettrail/pettrailbackend/mapper/PostEeMapper.java`
- `backend/src/main/resources/mapper/PostEeMapper.xml`
- `backend/src/main/java/com/pettrail/pettrailbackend/service/PostService.java`

**修复代码：**
```java
public boolean isUserEeLiked(Long postId, Long userId) {
    String userEeKey = "post:user:ee:" + postId + ":" + userId;
    Boolean exists = redisTemplate.hasKey(userEeKey);
    if (Boolean.TRUE.equals(exists)) {
        return true;
    }
    // Redis 未命中时，查数据库
    return postEeMapper.selectByPostIdAndUserId(postId, userId) != null;
}
```

### ✅ 中等问题修复（3个）

#### 4. 修复 `toggleLike` 方法 - 清除详情缓存
**问题：** 点赞/取消点赞后，详情接口中的点赞数更新延迟

**修复方案：**

```java
// 清除详情缓存
String cacheKey = "post:detail:" + postId;
redisTemplate.delete(cacheKey);
```

---

#### 5. 调整 `toggleEe` 和 `toggleLike` 查询顺序
**问题：** 先操作 Redis，再查询 Post。如果 postId 不存在，Redis 操作无效

**修复方案：**
- 将 Post 存在性检查提前到 Redis 操作之前
- 如果动态不存在，直接抛出异常，不执行 Redis 操作

**修复代码：**
```java
// 先检查动态是否存在
Post post = postMapper.selectById(postId);
if (post == null) {
    throw new NotFoundException("动态不存在");
}

// 然后再执行 Redis 操作
```

---

#### 6. 为 Redis Hash Key 设置 TTL
**问题：** `post:ee:{postId}` 和 `post:like:{postId}` 没有设置过期时间，存在内存泄漏风险

**修复方案：**
- 在每次操作 Hash 后设置 7 天过期时间

**修复代码：**
```java
redisTemplate.opsForHash().increment(eeKey, "count", 1);
redisTemplate.expire(eeKey, 7, TimeUnit.DAYS); // 设置 TTL
```

---

### ✅ 轻微问题修复（3个）

#### 7. 初始化 `eeCount` 字段
**问题：** `createPost` 方法中未初始化 `eeCount` 字段

**修复方案：**
- 在创建动态时显式初始化 `eeCount`

**修复代码：**
```java
post.setEeCount(0); // 初始化收藏计数
```

---

#### 8. 处理并发 `DuplicateKeyException`
**问题：** 高并发场景下可能出现重复点赞/收藏，导致未处理的异常

**修复方案：**
- 在 insert 操作周围添加 try-catch
- 捕获 `DuplicateKeyException` 并返回友好提示

**修复代码：**
```java
try {
    postEeMapper.insert(postEe);
} catch (DuplicateKeyException e) {
    log.warn("重复收藏: postId={}, userId={}", postId, userId);
    return true;
}
```

---

#### 9. 添加 `post_ee` 联合索引
**问题：** `selectCollectFeed` 查询缺少 `(user_id, post_id)` 联合索引

**修复方案：**
- 在 `post_ee` 表中添加联合索引 `idx_user_post (user_id, post_id)`

**修改文件：**
- `backend/src/main/resources/init.sql`

**SQL：**
```sql
KEY `idx_user_post` (`user_id`, `post_id`)
```

## 修复效果

### 修复前
- ❌ Redis 重启后收藏/点赞状态丢失
- ❌ 收藏/点赞数在详情接口中更新延迟 30 分钟
- ❌ 高并发时可能出现未处理异常
- ❌ Redis 内存持续增长无 TTL
- ❌ 收藏 Tab 显示状态不准确

### 修复后
- ✅ Redis 丢失时可自动从数据库恢复状态
- ✅ 收藏/点赞后立即更新详情数据（缓存失效）
- ✅ 并发操作时有完善的异常处理
- ✅ Redis Key 有 7 天 TTL，避免内存泄漏
- ✅ 收藏状态判断准确（Redis + 数据库双重校验）
- ✅ 收藏查询性能优化（新增联合索引）
