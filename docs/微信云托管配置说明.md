# 微信云托管配置说明

## 概述

本项目前端已配置微信云托管，微信小程序环境下将自动使用云托管服务调用后端 API。

## 配置信息

### 云托管环境

- **环境 ID**: `prod-3gyc3tpg28270da6`
- **服务名称**: `springboot-4fyd`
- **AppID**: `wxb210d75568abe1f7`

### 环境说明

| 环境 | 说明 | 配置值 |
|------|------|--------|
| 开发环境 (H5) | 本地开发，使用 HTTP 请求 | `http://localhost:8080` |
| 生产环境 (H5) | H5 生产环境 | `https://api.pettrail.com` |
| 微信小程序 | 使用微信云托管 | `cloud` (标记) |

## 代码改动说明

### 1. 环境配置文件 (`src/config/env.js`)

- 新增 `USE_CLOUD` 配置项，用于运行时判断是否使用云托管
- 优化配置项顺序，先定义云托管配置，再获取 API 基础地址
- 微信小程序环境下 `VITE_API_BASE_URL` 返回 `'cloud'` 标记

### 2. 请求封装 (`src/utils/request.js`)

- 新增 `ensureCloudInit()` 函数，检查云开发是否初始化
- 优化 `cloudRequest()` 函数：
  - 增加云开发支持检查
  - 增加详细的请求日志输出
  - 增加响应日志输出
  - 云开发不可用时降级为 HTTP 请求
- 优化请求拦截器 `request()` 函数：
  - 使用 `USE_CLOUD` 配置项判断
  - 微信小程序环境下优先使用云托管
  - 云开发不可用时降级为 HTTP 请求

### 3. 应用入口 (`src/main.js`)

- 完善云开发初始化日志
- 增加云托管服务名称输出
- 增加云托管模式启用提示
- 增加不支持云开发的警告提示
- 优化 `cloudRequest()` 函数（同 `request.js` 的改动）
- 优化 `request()` 函数判断逻辑（同 `request.js` 的改动）

## 请求流程

### 微信小程序环境

```
用户操作
   ↓
前端 API 调用
   ↓
request() 函数
   ↓
检查 USE_CLOUD && wx.cloud
   ↓ (true)
cloudRequest()
   ↓
wx.cloud.callContainer()
   ↓
云托管服务 (springboot-4fyd)
   ↓
Spring Boot 后端
   ↓
返回响应
```

### H5 环境

```
用户操作
   ↓
前端 API 调用
   ↓
request() 函数
   ↓
检查环境 (非微信小程序)
   ↓ (false)
httpRequest()
   ↓
uni.request()
   ↓
HTTP 请求 (localhost:8080 或 api.pettrail.com)
   ↓
返回响应
```

## 云托管请求示例

### 请求头

云托管请求会自动添加以下请求头：

```javascript
{
  'X-WX-SERVICE': 'springboot-4fyd',  // 云托管服务名称
  'Content-Type': 'application/json',
  'Authorization': 'Bearer <token>'    // 如果已登录
}
```

### 请求格式

```javascript
wx.cloud.callContainer({
  config: {
    env: 'prod-3gyc3tpg28270da6'  // 云托管环境 ID
  },
  path: '/api/users/login',        // API 路径
  header: {
    'X-WX-SERVICE': 'springboot-4fyd',
    'Content-Type': 'application/json'
  },
  method: 'POST',
  data: { code: 'xxx' }
})
```

## 注意事项

### 1. 云开发初始化

确保在 `main.js` 中已正确初始化云开发：

```javascript
// #ifdef MP-WEIXIN
if (wx.cloud) {
  wx.cloud.init({
    env: 'prod-3gyc3tpg28270da6',
    traceUser: true
  })
}
// #endif
```

### 2. 环境 ID 配置

确保 `project.config.json` 中的 `appid` 与云开发环境绑定的 AppID 一致。

### 3. 服务名称

`X-WX-SERVICE` 请求头必须正确设置，否则云托管无法路由请求。

### 4. 降级策略

当云开发不可用时，会自动降级为普通 HTTP 请求，确保应用可用性。

## 部署说明

### 前端部署

1. 构建微信小程序：
   ```bash
   npm run build:mp-weixin
   ```

2. 使用微信开发者工具打开 `dist/build/mp-weixin/` 目录

3. 上传代码到微信公众平台

### 后端部署

后端需要部署到微信云托管服务：

1. 在微信公众平台创建云托管环境
2. 创建服务 `springboot-4fyd`
3. 部署 Spring Boot 应用到云托管
4. 配置服务端口和路由

## 故障排查

### 问题 1: 云托管请求失败

**症状**: 控制台显示 "云托管请求失败"

**排查步骤**:
1. 检查云开发环境 ID 是否正确
2. 检查云托管服务名称是否正确
3. 检查云托管服务是否正常运行
4. 检查网络连接是否正常

### 问题 2: Token 无效

**症状**: 返回 401 错误

**排查步骤**:
1. 检查 Token 是否正确获取和存储
2. 检查 Token 是否过期
3. 检查后端 Token 验证逻辑

### 问题 3: 云开发未初始化

**症状**: 控制台显示 "当前环境不支持微信云开发"

**排查步骤**:
1. 确认在微信小程序环境下运行
2. 确认 `wx.cloud.init()` 已调用
3. 确认 `project.config.json` 配置正确
