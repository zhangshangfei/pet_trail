# 宠迹项目快速启动指南

> 最后更新：2026年4月27日

## 技术栈

### 后端
- Java 21
- Spring Boot 3.1.5
- MyBatis-Plus 3.5.7
- MySQL 8.0
- Redis 7.x
- 腾讯云 COS（文件存储）
- JWT（用户认证）
- RabbitMQ（延迟消息队列）

### 前端
- Uni-app 3.0（Vue 3）
- Pinia（状态管理）
- Sass（CSS预处理器）

### 管理后台
- Vue 3 + Element Plus + ECharts + Pinia

## 环境要求

- JDK 21+
- Maven 3.6+
- MySQL 8.0+
- Redis 7.0+
- Node.js 18+
- RabbitMQ 3.x+（可选，用于延迟消息提醒）

## 本地开发

### 1. 克隆项目

```bash
git clone https://github.com/zhangshangfei/pet_trail.git
cd pet-trail
```

### 2. 配置环境变量

创建 `backend/.env` 文件或直接修改 `application.yml`：

```env
# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USERNAME=root
DB_PASSWORD=root
DB_NAME=pet_trail

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379

# 腾讯云COS配置
TENCENT_COS_SECRET_ID=your-secret-id
TENCENT_COS_SECRET_KEY=your-secret-key
TENCENT_COS_BUCKET=your-bucket-name
TENCENT_COS_REGION=ap-guangzhou

# JWT配置
JWT_SECRET=your-secret-key-at-least-256-bits-long

# 前端URL
FRONTEND_URL=http://localhost:5173
```

### 3. 安装依赖

#### 后端

```bash
cd backend
mvn clean install
```

#### 前端

```bash
cd frontend
npm install
```

#### 管理后台

```bash
cd admin
npm install
```

### 4. 初始化数据库

```bash
mysql -u root -p < backend/src/main/resources/init.sql
```

### 5. 启动服务

#### 后端

```bash
cd backend
mvn spring-boot:run
```

后端服务将在 `http://localhost:8080` 启动

#### 前端（H5开发模式）

```bash
cd frontend
npm run dev
```

前端开发服务将在 `http://localhost:5173` 启动

#### 前端（微信小程序开发模式）

```bash
cd frontend
npm run dev:mp-weixin
```

然后在微信开发者工具中打开 `dist/dev/mp-weixin/` 目录

#### 管理后台

```bash
cd admin
npm run dev
```

管理后台将在 `http://localhost:5174` 启动

### 6. 测试

访问 Swagger API 文档：
```
http://localhost:8080/swagger-ui/index.html
```

测试健康检查接口：
```bash
curl http://localhost:8080/api/health
```

## Docker 部署

### 1. 配置环境变量

在项目根目录创建 `.env` 文件（参考 `.env.example`）：

```env
TENCENT_COS_SECRET_ID=your-secret-id
TENCENT_COS_SECRET_KEY=your-secret-key
JWT_SECRET=your-secret-key-at-least-256-bits-long
```

### 2. 启动所有服务

```bash
docker-compose up -d
```

这将启动：
- MySQL 8.0
- Redis 7
- RabbitMQ（延迟消息）
- Spring Boot 后端
- Nginx（反向代理）

### 3. 查看日志

```bash
docker-compose logs -f backend
docker-compose logs -f mysql
docker-compose logs -f redis
```

### 4. 停止服务

```bash
docker-compose down
```

### 5. 清理数据

```bash
docker-compose down -v
```

## 腾讯云COS配置

### 1. 创建COS Bucket

1. 登录腾讯云COS控制台
2. 创建存储桶，选择合适的地域
3. 设置访问权限为：公有读私有写（用于图片访问）
4. 配置跨域规则（CORS）

### 2. 配置CORS规则

```
来源: *
操作: GET, POST, PUT, DELETE, HEAD
Allow-Headers: *
Expose-Headers: ETag
超时: 0
```

### 3. 获取密钥

1. 登录腾讯云访问管理控制台
2. 创建API密钥
3. 保存SecretId和SecretKey

### 4. 配置到项目

将密钥配置到 `application.yml` 或环境变量中。

## 服务器部署

### 1. 上传代码到服务器

```bash
git clone https://github.com/zhangshangfei/pet_trail.git
cd pet-trail
```

### 2. 配置Nginx

编辑 `/etc/nginx/nginx.conf`：

```nginx
upstream backend {
    server localhost:8080;
}

server {
    listen 80;
    server_name your-domain.com;

    location /api/ {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
}
```

### 3. 配置SSL证书（可选）

使用Let's Encrypt免费证书：

```bash
sudo apt-get install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

### 4. 启动服务

```bash
sudo systemctl start mysql
sudo systemctl start redis
nohup java -jar backend/target/pet-trail-backend-1.0.0.jar > app.log 2>&1 &
sudo systemctl restart nginx
```

## 常见问题

### 1. 数据库连接失败

检查：
- MySQL是否启动
- 用户名密码是否正确
- 数据库是否创建

### 2. Redis连接失败

检查：
- Redis是否启动
- 端口是否正确

### 3. COS上传失败

检查：
- SecretId/SecretKey是否正确
- Bucket是否存在
- Bucket权限是否正确
- CORS规则是否配置

### 4. 端口被占用

修改 `application.yml` 中的 `server.port`

### 5. JWT验证失败

检查：
- JWT Secret是否一致
- Token是否过期
