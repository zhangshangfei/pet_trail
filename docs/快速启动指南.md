# 宠迹项目快速启动指南

## 技术栈

### 后端
- Java 21
- Spring Boot 3.2.0
- MyBatis-Plus 3.5.5
- MySQL 8.0
- Redis 7
- 阿里云 OSS（文件存储）
- JWT（用户认证）

### 前端
- Uni-app
- uView UI
- Pinia
- ECharts

## 环境要求

- JDK 21+
- Maven 3.6+
- MySQL 8.0+
- Redis 6.0+
- Node.js 16+

## 本地开发

### 1. 克隆项目

```bash
cd G:\openclaw_project\pet-trail
```

### 2. 配置环境变量

创建 `backend/.env` 文件：

```env
# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USERNAME=root
DB_PASSWORD=root
DB_NAME=pet_trail

# Redis配置
REDIS_HOST=localhost
REDIS_PORT=6379

# 阿里云OSS配置（需要先创建OSS Bucket）
ALIYUN_OSS_ACCESS_KEY_ID=your-access-key-id
ALIYUN_OSS_ACCESS_KEY_SECRET=your-access-key-secret

# JWT配置
JWT_SECRET=your-secret-key-at-least-256-bits-long

# 前端URL
FRONTEND_URL=http://localhost:5173
```

### 3. 安装依赖

#### 后端

```bash
cd backend
mvn clean install
```

#### 前端

```bash
cd frontend
npm install
```

### 4. 初始化数据库

```bash
# 创建数据库和表
mysql -u root -p < backend/init.sql
```

### 5. 启动服务

#### 后端

```bash
cd backend
mvn spring-boot:run
```

后端服务将在 `http://localhost:8080` 启动

#### 前端

```bash
cd frontend
npm run dev
```

前端开发服务将在 `http://localhost:5173` 启动

### 6. 测试

访问 Swagger API 文档：
```
http://localhost:8080/swagger-ui/index.html
```

测试健康检查接口：
```bash
curl http://localhost:8080/api/health
```

## Docker 部署

### 1. 配置环境变量

在项目根目录创建 `.env` 文件：

```env
ALIYUN_OSS_ACCESS_KEY_ID=your-access-key-id
ALIYUN_OSS_ACCESS_KEY_SECRET=your-access-key-secret
JWT_SECRET=your-secret-key-at-least-256-bits-long
```

### 2. 启动所有服务

```bash
docker-compose up -d
```

这将启动：
- MySQL 8.0
- Redis 7
- Spring Boot 后端
- Nginx（反向代理）

### 3. 查看日志

```bash
docker-compose logs -f backend
docker-compose logs -f mysql
docker-compose logs -f redis
```

### 4. 停止服务

```bash
docker-compose down
```

### 5. 清理数据

```bash
docker-compose down -v
```

## 阿里云OSS配置

### 1. 创建OSS Bucket

1. 登录阿里云OSS控制台
2. 创建Bucket，名称为 `pet-trail-bucket`
3. 设置访问权限为：公共读（用于图片访问）
4. 配置跨域规则（CORS）

### 2. 配置CORS规则

```
来源: *
允许Methods: GET, POST, PUT, DELETE, HEAD
允许Headers: *
暴露Headers: ETag
缓存时间: 0
```

### 3. 获取AccessKey

1. 登录阿里云RAM控制台
2. 创建RAM用户
3. 为用户授予 `AliyunOSSFullAccess` 或自定义权限
4. 保存AccessKey ID和Secret

### 4. 配置到项目

将AccessKey配置到 `backend/.env` 或 `docker-compose.yml` 中的环境变量。

## 阿里云ECS部署

### 1. 上传代码到服务器

```bash
# 使用Git
git clone <your-repo-url>
cd pet-trail

# 或者使用SCP上传
scp -r backend/ root@your-server-ip:/opt/pet-trail/
```

### 2. 配置Nginx

编辑 `/etc/nginx/nginx.conf`：

```nginx
upstream backend {
    server localhost:8080;
}

server {
    listen 80;
    server_name your-domain.com;

    location /api/ {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
}
```

### 3. 配置SSL证书（可选）

使用Let's Encrypt免费证书：

```bash
# 安装certbot
sudo apt-get install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com
```

### 4. 配置防火墙

```bash
# 开放端口
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

### 5. 启动服务

```bash
# 启动MySQL
sudo systemctl start mysql

# 启动Redis
sudo systemctl start redis

# 启动后端（使用Systemd或直接运行jar）
nohup java -jar backend/target/pet-trail-backend-1.0.0.jar > app.log 2>&1 &

# 重启Nginx
sudo systemctl restart nginx
```

## 常见问题

### 1. 数据库连接失败

检查：
- MySQL是否启动
- 用户名密码是否正确
- 数据库是否创建

### 2. Redis连接失败

检查：
- Redis是否启动
- 端口是否正确

### 3. OSS上传失败

检查：
- AccessKey是否正确
- Bucket是否存在
- Bucket权限是否正确
- CORS规则是否配置

### 4. 端口被占用

修改 `application.yml` 中的 `server.port`

### 5. JWT验证失败

检查：
- JWT Secret是否一致
- Token是否过期
