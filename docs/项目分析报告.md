## 二、尚未实现的功能（按优先级排序）

### 🔴 P0 - 核心缺失

#### 1. 动态推荐Tab没有真正的推荐逻辑
- PostService.java 中 recommend Tab 直接调用 selectFeed，与"全部"Tab完全一样
- 用户推荐算法已实现（RecommendService），但动态推荐算法未实现
- 应基于用户兴趣、互动历史、热度衰减等维度推荐动态

#### 2. PostMapper.xml ResultMap 字段缺失
- BaseResultMap 缺少 location、stickers、bubble、ee_count、videos 字段映射
- 通过 MyBatis XML 查询返回的 Post 对象这些字段始终为 null
- 虽然 selectById 能自动映射，但 XML 定义的查询可能受影响

#### 3. 动态详情页缺少贴纸/气泡/位置展示
- post/detail.vue 没有展示 stickers、bubble、location
- 首页已实现但详情页遗漏，用户体验不一致

#### 4. 用户详情页动态列表缺少贴纸/气泡/位置
- user/detail.vue 的动态卡片同样缺少这些字段展示

### 🟡 P1 - 重要功能

#### 5. 动态删除功能
- 后端无 DELETE /api/posts/{id} 接口
- 前端无删除入口
- 用户发布错误动态后无法删除

#### 6. 评论删除功能
- 后端无删除评论接口
- 评论者或动态作者无法删除不当评论

#### 7. 举报功能
- 无动态举报/用户举报接口
- 社区内容安全缺乏用户参与机制

#### 8. 内容审核是空壳
- ContentAuditService 只有占位敏感词，auditImage 直接返回 true
- 发布动态时甚至没有调用审核服务

#### 9. 成就系统后端未实现
- 数据库有 achievements 和 user_achievements 表
- 但后端没有 Entity、Mapper、Service、Controller
- 前端也没有成就展示页面

#### 10. 挑战赛话题后端未实现
- publish/index.vue 挑战赛列表是前端硬编码的4条数据
- 后端无 Challenge 实体、表、接口
- 发布动态时 challengeTag 字段未真正保存到数据库

#### 11. 全局搜索功能缺失
- 只有发现页的用户搜索
- 没有动态内容搜索、宠物品种搜索
- 顶部搜索图标点击跳转到发现页，但缺少独立的搜索页面

### 🟢 P2 - 体验优化

#### 12. 关于/反馈/设置页面未实现
- me/index.vue 三个入口都是 showToast("未实现")
- 设置页应包含：账号管理、隐私设置、通知偏好、清除缓存等

#### 13. 通知实时推送
- 当前通知是被动拉取（用户打开通知页才加载）
- 缺少 WebSocket/轮询机制实时获取未读数
- UserTopBar 的未读数不会自动更新

#### 14. 数据库迁移自动化
- 没有 Flyway/Liquibase
- 新增字段需要手动执行 ALTER TABLE
- 部署时容易遗漏

#### 15. 并发关注异常处理
- FollowService.toggleFollow() 没有捕获 DuplicateKeyException
- 高并发下可能抛出数据库异常

## 三、性能优化点

### 🔴 高优先级

#### 1. CommentService N+1 查询问题
- CommentService.convertToVO() 每条评论单独查询用户信息
- 每条回复也单独查询用户信息
- 20条评论可能产生 40+ 次数据库查询
- 建议：批量预加载用户信息（与 PostController 的 convertToPostVOList 同样模式）

#### 2. RecommendService 全量加载问题
- RecommendService.computeRecommendations() 查询所有未关注用户计算评分
- 用户量大时内存和性能问题严重
- 建议：限制候选集大小（如取活跃用户 Top 500），或使用分页计算

#### 3. PostMapper.xml ResultMap 字段缺失
- 缺少 videos、ee_count、location、stickers、bubble 映射
- 可能导致部分查询场景数据丢失

### 🟡 中优先级

#### 4. 图片上传未压缩
- 发布动态时直接上传原图
- 应在上传前进行压缩（尤其是手机拍摄的高分辨率照片）
- 建议：使用 uni.compressImage 压缩后再上传

#### 5. 通知列表无分页优化
- NotificationService 使用 LIMIT offset,size 分页
- 数据量大时深分页性能差
- 建议：使用游标分页（基于 last_id）

#### 6. 缺少 API 限流
- 发布动态、点赞、评论等接口无频率限制
- 恶意用户可刷接口
- 建议：使用 Redis + 令牌桶/滑动窗口限流

#### 7. 前端请求缓存粒度粗
- 当前 3 秒 TTL 的全局缓存
- 对实时性要求高的接口（点赞状态）不应缓存
- 建议：按接口类型设置不同缓存策略

### 🟢 低优先级

#### 8. 前端无全局错误处理
- 各页面独立 try-catch
- 建议：统一请求拦截器处理 401/403/500

#### 9. 缺少骨架屏
- 列表加载时只有"加载中..."文字
- 建议：添加骨架屏提升感知速度

#### 10. 无单元测试
- 后端无任何测试类
- 建议：至少为 Service 层核心逻辑添加测试
